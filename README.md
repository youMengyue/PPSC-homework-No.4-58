# Демонстрация производительности инструкций SIMD

## Номер студента: 58

---

### 1. Выбор инструкции на основе номера

В соответствии с заданием, инструкция SIMD для демонстрации была определена на основе номера студента `58`:

*   **Префикс инструкции**:
    *   `58 % 2 = 0` (остаток от деления 58 на 2).
    *   Согласно правилу, выбран префикс `_mm`.

*   **Суффикс инструкции**:
    *   `58 % 4 = 2` (остаток от деления 58 на 4).
    *   Согласно правилу, выбран суффикс `_pd` (Packed Double — упакованные числа двойной точности).

*   **Ядро инструкции**:
    *   `58 / 8 = 7` (целая часть от деления 58 на 8).
    *   Согласно правилу, выбрана операция `_div` (Division — деление).

---

### 2. Итоговая инструкция: `_mm_div_pd`

Комбинация трёх частей даёт итоговую инструкцию для выполнения задания: **`_mm_div_pd`**.

Эта инструкция является частью набора SSE2 и выполняет параллельное деление двух пар чисел с плавающей запятой двойной точности (`double`), упакованных в 128-битные регистры.

---

### 3. Цель программы

Цель данной программы — наглядно продемонстрировать практическую пользу инструкции `_mm_div_pd` через сравнение производительности.

Программа выполняет следующие шаги:
1.  Создаёт два больших массива, заполненных случайными числами типа `double` (делимое и делитель).
2.  **Метод 1 (Скалярные вычисления)**: Вычисляет частное двух массивов с помощью стандартного цикла `for`, обрабатывая элементы последовательно. Время выполнения замеряется.
3.  **Метод 2 (SIMD вычисления)**: Вычисляет частное тех же массивов с использованием инструкции `_mm_div_pd`, которая обрабатывает по два элемента `double` за одну операцию. Время выполнения также замеряется.
4.  В конце программа выводит результаты замеров и рассчитывает, во сколько раз SIMD-версия оказалась быстрее скалярной, что наглядно показывает преимущество параллельной обработки данных.

---

### 4. Как скомпилировать и запустить

Проект использует `Makefile` для управления сборкой. Убедитесь, что у вас установлены `g++` и `make`.

1.  **Сборка проекта**:
    Откройте терминал, перейдите в корневую папку проекта и выполните команду:
    ```bash
    make
    ```
    Эта команда скомпилирует исходный код с флагом оптимизации `-O3`, что крайне важно для корректного теста производительности, и создаст исполняемый файл `homework_58`.

2.  **Запуск программы**:
    Для запуска теста производительности выполните:
    ```bash
    make run
    ```
    Или можно запустить файл напрямую:
    ```bash
    ./homework_58
    ```

3.  **Очистка**:
    Чтобы удалить скомпилированный файл, используйте команду:
    ```bash
    make clean
    ```
